# Practice 1 — Record definition and dataset schema

## Topic

EGFR kinase inhibitors dataset (wild-type and mutant variants).

## Scientific task

Collect and standardize experimentally reported EGFR kinase inhibitor measurements (molecules, mutational targets, inhibitor activity values, assay conditions) to compare potency, selectivity, and support predictive model benchmarking.

## One-record definition

**One record** = one experimentally reported quantitative bioactivity measurement (IC50, Ki, Kd, EC50) of a specific small-molecule inhibitor tested against a specific human EGFR variant (wild-type or designated mutant) in an in vitro biochemical or cellular assay from a single literature or database source (one row in `data/processed/dataset.csv`).

## Examples of records

| Example | Why it counts |
|---------|----------------|
| IC50 = 12 nM for compound Erlotinib against EGFR L858R/T790M mutant from Table 3 in Zhang 2020 | Single measurement + compound name + target variant + source |
| Ki = 1.5 nM from supplementary Excel table for a novel covalent inhibitor vs EGFR WT in kinase assay | One numeric binding outcome tied to a specific compund-variant pair under defined *in vitro* biochemical conditions  |

## Non-record examples

| Example | Why it is not a record |
|---------|-------------------------|
| General review of EGFR mutation-induced resistance pathways without compound-specific quantitative assay measurements | No measurement |
| A high-throughput screening plate map listing 100 compound structures as "active" against EGFR without reporting exact numerical IC50/EC50 values | Represents a qualitative (rather than quantitative) bioactivity classification |
| Predicted binding affinity generated by a deep learning model for 50 drug candidates against EGFR L858R | In silico predictions are out of scope; the dataset must contain only experimentally verified values |
| In vivo tumor reduction percentage (e.g., "75% tumor shrinkage") in mice treated with Gefitinib | In vivo pharmacodynamics and pharmacokinetics metrics are excluded to keep the target-binding benchmark clean |

## Dataset fields

The dataset schema consists of both required fields (necessary for a valid entry) and optional metadata fields (retained to preserve assay context).

### Required Fields
1. `record_id` (string): Stable unique identifier for one binding measurement record.
2. `compound_id` (string): Unique identifier for the small molecule (e.g., internal ID or database ID like `CHEMBL553`).
3. `compound_smiles` (string): Standardized canonical 2D SMILES string representing the chemical structure.
4. `egfr_variant` (string): Target mutational status (e.g., `WT`, `L858R`, `T790M`, `L858R_T790M`, `Exon19_del`).
5. `standard_type` (string): The reported metric; must be one of `["IC50", "Ki", "Kd", "EC50"]`.
6. `standard_value` (numeric): The numerical result of the assay (must be $\ge 0$).
7. `standard_units` (string): The measurement units `["nM"]`.
8. `source_id` (string): Identifier linking to an entry in `specs/source_map.json`.

### Optional Fields
9. `compound_name` (string): Generic or common name (e.g., `Erlotinib`, `Gefitinib`).
10. `standard_relation` (string): Precision relation operator; must be one of `["=", "<", ">", "<=", ">="]`.
11. `pchembl_value` (numeric): Standardized negative log10 of the molar activity value.
12. `assay_type` (string): Context of the assay; must be one of `["Biochemical", "Cellular"]`.
13. `cell_line` (string): Cell line name used for cellular assays (e.g., `Ba/F3`, `H1975`, `PC-9`).
14. `atp_concentration_uM` (numeric): ATP concentration in micromolar (uM) used in biochemical assays.
15. `covalent_flag` (boolean): Flag indicating if the compound is designed to covalently bind to C797 or other residues.
16. `extraction_method` (string): How the record was obtained (pdf_manual, pdf_table, web_scrape, etc.).
17. `extraction_confidence` (string): Confidence in extracted values: high, medium, low, or unknown.
18. `notes` (string): Free-text notes, caveats, or provenance details.

## Ambiguous cases

During dataset curation, several edge cases can introduce noise. We establish the following handling rules:

- **Multiple assays for the same compound-target variant:** 
   We preserve these as separate records (rows) in the final dataset instead of averaging them. This retains vital metadata such as different cell lines (`cell_line`), ATP concentrations (`atp_concentration_uM`), and lab-to-lab variance, which helps machine learning models learn assay-dependent bias.
- **Activity reported as a range (e.g., "IC50 = 10–50 nM"):**
   We will represent the `standard_value` as `null` or record the mid-point value while setting a flag, depending on modeling requirements. If strict quantitative regression is the downstream goal, these records should be excluded from the clean training set.
- **Missing stereochemical descriptors in publications:**
   If a compound is tested as a racemic mixture or the paper does not specify stereocenters, the flat, non-chiral 2D SMILES will be generated. 
- **Duplicate compound-variant records in different databases:**
   When merging ChEMBL and BindingDB, duplicate entries originating from the same paper will be deduplicated based on identical canonical SMILES, `egfr_variant`, `standard_value`, and `source_id`.

## Validation rules

The validation pipeline (running via `specs/validation_rules.json` and checking outputs) will enforce:
- **Structural Integrity:** Every SMILES string under `compound_smiles` must be valid and parsable by RDKit.
- **Value Constraints:** `standard_value` and `atp_concentration_uM` must be non-negative real numbers.
- **Categorical Integrity:** `assay_type`, `standard_relation`, `standard_type`, and `standard_units` must strictly belong to their respective allowed enum sets.
- **Deduplication Check:** No two rows can share the exact same combination of `compound_smiles`, `egfr_variant`, `standard_type`, `standard_value`, and `source_id`.
